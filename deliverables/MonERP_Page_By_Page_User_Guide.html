1. Гарын авлагыг хэрхэн ашиглах вэ

Зорилго: Энэ гарын авлага нь MonERP системийг хоёр өөр хэрэглэгчийн өнцгөөс тайлбарлана: Admin/HR/Менежер эрхтэй хэрэглэгч болон ажилтан/user эрхтэй хэрэглэгч. Хуудас бүр дээр юу харах, ямар дарааллаар ажиллах, ямар алдаа гарахаас сэргийлэхийг нэг дор цэгцэлсэн.
ХэрэглэгчҮндсэн үүрэгЭнэ гарын авлагаас унших хэсэг
Admin / HR / МенежерАжилтан, эрх, ирц, хүсэлт, цалин, баримт, тайлан, тохиргоог удирдана.2-26-р хэсэг, 35-39-р checklist
Ажилтан / UserӨөрийн ирц, ээлж, хүсэлт, цалин, мэдээ, баримт, хувийн мэдээллээ харна.27-34-р хэсэг, 40-р checklist
Супер админ / Техник хариуцагчTenant, session, production тохиргоо, audit, backup, эрхийн бодлого хянана.24-26, 37-41-р хэсэг
Санах гол дараалал: Ажилтан бүртгэлтэй болно -> эрх онооно -> ирц/ээлж үүснэ -> хүсэлт батлагдана -> цалин бодогдоно -> санхүү/тайлан/баримт дээр хянана.

2. Admin dashboard - өглөө бүр эхлэх самбар

Route /

Admin хэрэглэгч системд орсны дараа хамгийн түрүүнд хяналтын самбар дээр байгууллагын тухайн өдрийн байдал, шийдвэрлэх зүйл, санхүүгийн дохио, ажилтны идэвхийг харна. Энэ хуудас нь бүртгэл хийхээс илүү юу түрүүлж хийх вэ гэдгийг хэлж өгдөг төв самбар юм.

Карт / widgetЮуг илэрхийлэх вэAdmin юу хийх вэАнхаарах зүйл
Нийт ажилтанИдэвхтэй ажилтны тоо.Шинэ ажилтан орсон бол Employees руу орж бүртгэл, хэлтэс, албан тушаал, эрхийг шалгана.Хуучин ажилтны төлөвийг идэвхгүй болгоогүй бол тоо буруу өснө.
Ирцийн хувьӨнөөдрийн ирсэн/хоцорсон/тасалсан байдлын хувь.Attendance эсвэл Roster руу орж хоцролт, таслалт, гараагүй бүртгэлийг шалгана.Ээлжийн хуваарь үүсээгүй бол ирцийн хувь буруу харагдаж болно.

Admin өглөөний 5 алхам: 1) Хүлээгдэж буй хүсэлт шалгах. 2) Ирцийн хувь шалгах. 3) Action Center хоосон эсэхийг үзэх. 4) Шинэ ажилтан/өөрчлөлт activity дээр байгаа эсэхийг харах. 5) Мөнгөн урсгал болон цалингийн төсвийн анхааруулгыг шалгах.

3. Dashboard дээрх хурдан үйлдлүүд

Хяналтын самбарын баруун дээд хэсэгт хугацаа сонгох, туршилтын өгөгдөл, нэхэмжлэх үүсгэх, ажилтан нэмэх, чөлөө хүсэх, товч харагдац зэрэг command товчнууд байна.

ҮйлдэлХэзээ хэрэглэх вэҮр дүн

Туршилтын өгөгдөлDemo орчинд тест өгөгдөл харах шаардлагатай үед.Production дээр андуурч асаахгүй.
Нэхэмжлэх үүсгэхСанхүүгийн баримт хурдан эхлүүлэх үед.Invoice/Finance flow руу шилжинэ.
Ажилтан нэмэхШинэ ажилтан бүртгэх үед.Employees form нээгдэнэ.
Чөлөө хүсэхAdmin өөрөө эсвэл ажилтны өмнөөс хүсэлт эхлүүлэх үед.Request workflow эхэлнэ.
Товч харагдацЖижиг дэлгэц эсвэл хурдан хяналт хийх үед.Widget-үүд шахагдсан байдлаар гарна.
Mobile анхаарах: Утасны дэлгэц дээр картууд доошоо дараалж харагдана. Хэрэв мөр тасрах, товч багтахгүй, огноо урт хэлбэрээр гарах асуудал гарвал browser cache цэвэрлэж шинэ хувилбар ачаалсан эсэхийг шалгана.

ҮйлдэлДараалалЗаавал шалгах талбар

Нормын хувцасны модуль нь 3 өөр үүрэгтэй ажиллана: Admin/HR норм, хувцасны төрөл, ажилтанд олгох эрхийг үүсгэнэ; нярав/warehouse ажилтанд бодитоор хувцсыг хүлээлгэн өгч "Авсан" төлөв болгоно; ажилтан/user өөрт олгогдсон болон авсан хувцсаа харна.

Алхам 1. Хувцасны төрөл болон норм тохиргоо үүсгэх

Хэн хийх вэ: Admin эсвэл HR. Цэс: Нормын хувцас -> Норм тохиргоо. Эрх: workwear.approve.

  1. Sidebar-аас Нормын хувцас цэс рүү орно.
  2. Норм тохиргоо товч дарж хувцасны төрөл, ангилал, жилийн норм, үнэ, идэвхтэй төлөвийг тохируулна.
  3. Жишээ: Ажлын гутал - жилд 1ш, Малгай - жилд 1ш, Бээлий - жилд 2ш гэх мэт.
  4. Нормын загвар хэрэгтэй бол албан тушаал/ажлын байртай холбож template үүсгэнэ.
Анхаарах: Энэ хэсгийг нярав хийхгүй. Нярав зөвхөн бэлэн үүссэн эрхийн дагуу хувцас хүлээлгэн өгнө. Норм, загвар, олгох эрх үүсгэх нь HR/Admin-ийн ажил.

Алхам 2. Ажилтанд хувцасны эрх үүсгэх

Хэн хийх вэ: Admin эсвэл HR. Цэс: Нормын хувцас -> Олгох эсвэл Бөөнөөр. Эрх: workwear.approve.

  1. Нормын хувцас хуудсанд Олгох товч дарна.
  2. Ажилтан, хувцасны төрөл, тоо ширхэг, хэмжээ, жил, дуусах хугацаа, тэмдэглэл оруулна.
  3. Олон ажилтанд зэрэг эрх үүсгэх бол Бөөнөөр товч ашиглаж ажилтнууд сонгоно.
  4. Систем тухайн жилийн норм давсан эсэхийг шалгана. Давсан бол тухайн ажилтны мөр алгасагдана.
  5. Амжилттай хадгалахад төлөв нь Эрх үүссэн / granted болно.
Эрх үүсгэх checklist: зөв ажилтан, зөв хувцас, зөв тоо ширхэг, хэмжээ, жил, дуусах хугацаа, норм давсан эсэх, тайлбар.

Алхам 3. Нярав хувцсыг бодитоор олгох

Хэн хийх вэ: Warehouse/нярав, Admin эсвэл HR. Цэс: Хувцас олгох /warehouse/workwear. Эрх: workwear.write.

  1. Нярав өөрийн эрхээр нэвтэрч sidebar-аас Хувцас олгох цэс рүү орно.
  2. Ажилтны нэр, код, дугаараар хайж ажилтнаа сонгоно.
  3. Сонгосон ажилтанд HR/Admin-аас үүсгэсэн Эрх үүссэн төлөвтэй хувцаснууд гарна.
  4. Өгөх хувцсаа сонгож, шаардлагатай бол size/хэмжээ оруулна.
  5. Олгох товч дарснаар төлөв Авсан / collected болно.
  6. Авсан огноо, олгосон хэрэглэгч, хэмжээ систем дээр хадгалагдана.
Чухал ялгаа: HR/Admin "эрх үүсгэнэ"; нярав "барааг гар дээр өгч авсан төлөв болгоно". Эрх үүсээгүй хувцсыг нярав шууд олгох ёсгүй.

Алхам 4. Ажилтан өөрийн хувцсаа харах

Хэн хийх вэ: Ажилтан/User. Цэс: Миний хувцас /me/workwear. Эрх: workwear.read.

  1. Ажилтан sidebar-аас Миний хувцас руу орно.
  2. Хүлээгдэж буй хэсгээс одоогоор эрх нь үүссэн боловч аваагүй хувцсаа харна.
  3. Түүх хэсгээс авсан, хугацаа дууссан хувцасны мэдээллээ харна.
  4. Мэдээлэл буруу байвал HR эсвэл няравт мэдэгдэнэ.

35. Admin өдөр тутмын checklist

  1. Dashboard дээр хүлээгдэж буй хүсэлт, ирцийн хувь, Action Center шалгах.
  2. Attendance дээр хоцролт, таслалт, гараагүй бүртгэлийг цэвэрлэх.
  3. Requests дээр pending хүсэлтүүдийг шийдэх.
  4. Employees дээр шинэ/гарсан ажилтны төлөв, эрх, хэлтэс зөв эсэхийг шалгах.
  5. News/Documents дээр яаралтай мэдэгдэл, албан бичиг байгаа эсэхийг шалгах.
  6. Finance/Invoices дээр хугацаа хэтэрсэн төлбөр, нэхэмжлэхийг харах.
  7. Audit дээр сэжигтэй эрх, цалин, санхүүгийн өөрчлөлт байгаа эсэхийг шалгах.

36. Admin сарын хаалтын checklist

  1. Сарын бүх roster батлагдсан эсэхийг шалгах.
  2. Ирцийн дутуу check-out, таслалт, чөлөөний хүсэлтийг цэвэрлэх.
  3. Урьдчилгаа, нэмэгдэл, суутгал бүрийг баталгаажуулах.
  4. Payroll бодож, sample ажилтнууд дээр дүн тулгах.
  5. Цалингийн тайлан, татварын задаргаа, гар дээр авах дүнг шалгах.
  6. Documents дээр сарын тушаал, акт, тайлан, хавсралтыг архивлах.
  7. Backup амжилттай хийгдсэн эсэхийг шалгах.
